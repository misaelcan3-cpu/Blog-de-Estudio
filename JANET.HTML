Introducción

La verificación y validación (V&V) son procesos fundamentales para garantizar que una aplicación web cumpla con especificaciones técnicas y expectativas del usuario.

5.1 Planificación de la Verificación y Validación (V&V)

Resumen: La planificación de V&V define cómo se evaluará la calidad de una webapp durante todo su ciclo de vida. Incluye preparación, responsables, herramientas y métricas para garantizar que el producto final cumpla con los estándares y necesidades del usuario.

Contenido principal

  • Definición: Verificación revisa artefactos internos (código, documentos). Validación confirma que el sistema cumple lo que el usuario espera.
  • Objetivo: La planificación de V&V tiene como finalidad:
    • Definir qué se evaluará, cómo, cuándo y con qué recursos.
    • Garantizar que el software cumpla requisitos funcionales y no funcionales.
    • Reducir riesgos, defectos, sobrecostos y retrabajos.
    • Establecer estándares que aseguren calidad desde las primeras etapas.
  • Elementos del plan:
    • Responsables
    • Actividades
    • Metricas de calidad
    responsables, actividades, entregables, métricas de calidad.
  • Herramientas: Gantt, Trello, Jira, GitHub Projects, hojas de control.
  • Tip práctico: define métricas claras (ej. tiempo de carga < 2s, 0 defectos críticos en producción).

5.2 Inspección de Software Basado en Web

Resumen: La inspección es una revisión formal y sistemática para encontrar defectos en artefactos: código, requisitos, diseño o documentación. Se realiza mediante revisiones, walkthroughs o auditorías y se apoya en checklists y herramientas colaborativas.

Contenido principal

  • ¿Qué es una inspección? Actividad formal para detectar defectos antes de avanzar de etapa.
  • Tipos:
    • Revison:Revisión informal o semiformal del código, interfaz o documentos.
    • Walkthrough:El autor del software presenta el diseño o código a un grupo para analizarlo y detectar fallos.
    • Auditoria:Evaluación formal realizada por un tercero, cuyo objetivo es verificar el cumplimiento de estándares, normativas o metodologías.
  • Checklist típico: contenido, enlaces, navegación, interfaz, seguridad, estándares.
  • Ejemplo: Se inspecciona la pgina principal de inicion de sesion de la pagina para encontrar que se valide el correo u otros campos que necesiten validacion.
  • Herramientas: GitHub Issues, Jira, Trello.

5.3 Pruebas de Aplicaciones Web

Resumen: Las pruebas permiten identificar errores en funcionalidad, seguridad, desempeño, usabilidad y compatibilidad. Requieren planificación y generación de evidencias (capturas, logs, reportes).

Contenido principal

  • Definición: ejecución controlada de la webapp para encontrar errores.
  • Tipos:
    • Pruebas Funcionales:
      • Probar el inicio de sesión con combinaciones válidas e inválidas
      • Verificar que los enlaces y botones ejecuten la acción correcta.
      • Validar que los formularios envíen y reciban datos apropiadamente.
    • Pruebas No Funcionales:
      • Medir tiempo de carga de la página principal.
      • Revisar el uso de CPU/RAM en el navegador.
      • Verificar estabilidad bajo múltiples solicitudes simultáneas.
    • Pruebas de Funcionalidad:
      • Comprobar que un usuario pueda completar una tarea sin instrucciones.
      • Evaluar claridad de textos, colores, botones y mensajes.
      • Probar accesibilidad con teclado y lectores de pantalla.
    • Pruebas de Seguridad:
      • Intentar inyección SQL en formularios.
      • Verificar protección contra XSS reflejado y almacenado.
      • Comprobar uso de HTTPS y políticas de seguridad (CSP).
    • Pruebas de Compatibilidad:
      • Probar interfaz en varios navegadores y tamaños de pantalla.
      • Revisar comportamiento del diseño responsive.
  • Plan de pruebas: casos, datos, responsables, ambiente.
  • Evidencias: capturas, reportes, logs, videos. Guardarlas da trazabilidad.

5.4 Ejemplos de Pruebas para Webapps

Resumen: Casos prácticos (formulario, login, carrito) con datos, resultado esperado y evidencia. Sirven para práctica en laboratorio o como plantillas para pruebas reales.

Ejemplos (detallados)

  1. Formulario de registro
    • Caso: verificar campos obligatorios
    • Datos: nombre vacío, correo inválido
    • Resultado esperado: mensajes de validación
    • Evidencia: pantalla de error
  2. Login de usuario
    • Caso: login correcto / incorrecto
    • Datos: usuario válido / usuario no registrado
    • Resultado esperado: acceso o alerta
    • Evidencia: captura de dashboard o error
  3. Carrito de compras
    • Caso: agregar productos y actualizar total
    • Datos: cantidades 1–5
    • Resultado esperado: cambio correcto en el total
    • Evidencia: imagen del carrito

5.5 Implementación de Aplicaciones Web

Resumen: La implementación transforma diseños y requisitos en código. Incluye control de versiones, CI/CD y buenas prácticas; está estrechamente ligada a la V&V porque cada entrega debe ser verificable.

Contenido principal

  • Relación implementación–V&V: cada módulo debe ser probado antes de integrarse.
  • Control de versiones: Git, ramas, commits, pull requests.
  • CI/CD: Jenkins, GitHub Actions, GitLab CI.
  • Buenas prácticas: modularidad, documentación, pruebas unitarias.
  • Errores comunes: mala gestión de versiones, falta de pruebas, dependencias rotas.

5.6 Evaluación de Aplicaciones Web

Resumen: La evaluación revisa rendimiento, seguridad, accesibilidad, SEO y compatibilidad. Se usan heurísticas y matrices con puntajes para tomar decisiones de mejora.

Contenido principal

  • Evaluación integral: funcionalidad, estructura, experiencia del usuario.
  • Heurísticas: Nielsen, diseño responsive, WCAG.
  • Criterios técnicos: rendimiento, seguridad, SEO, tiempos de carga.
  • Matriz de evaluación con puntajes para priorizar mejoras.

5.7 Mantenimiento de Aplicaciones Web

Resumen: El mantenimiento modifica y actualiza la aplicación tras su entrega para corregir fallos, mejorar el rendimiento o agregar funcionalidades. Es continuo y requiere procesos de control de incidencias y monitoreo.

Contenido principal

  • Tipos: correctivo, preventivo, adaptativo, perfectivo.
  • Estrategias: logs, monitoreo, actualizaciones, respaldos.
  • Control de incidencias: tickets, prioridades, SLA.
  • Ejemplos: actualización de librerías, corrección de bugs, cambios en diseño.
